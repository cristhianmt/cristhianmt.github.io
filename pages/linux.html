Linea de comandos Linux

  • File system
    1. /: equivalente a C:.
    2. /root: directorio con todos los privilegios del usuario.
    3. /etc: contiene generalmente los archivos de configuración, cuando y como inician los programas.
      1. /passwd: almacena la información del usuario, se usa para ver todos los usuarios en un sistema.
      2. /shadow : tiene todas las contraseñas de todos los usuarios.
    4. /home: directorio principal del usuario.
    5. /mnt: directorio donde se montan archivos manualmente.
    6. /media: se suelen montar CD y dispositivos extraibles como USB.
    7. /bin: directorio de aplicaciones binarias importantes (el equivalente a ejecutables en Microsoft).
    8. /lib: directorio donde se encuentran todas las librerias del sistema (programas que son similares a los DLL de Microsoft).
    9. /proc: directorio que ofrecer información sobre el sistema cerca de los procesos.
    10. /sys: contiene archivos del sistema.
    11. /usr: contiene todos los archivos ejecutables.
    12. sudo -l: imprime los comandos permitidos por los usuarios actuales.
    13. Nano /etc/sudoers : dar permisos de ejecución de administrador.
      Debajo de root poner:
      <user> ALL=(ALL:ALL) ALL

  • Permite buscar cualquier archivo en el sistema. 
    find
    1. /: enumera todos los archivos del sistema operativo.
    2. /<name directory>: enumera todos los archivos del directorio especifico.
    3. -type <type>: enumerar el tipo de archivo.
      1. d: búsqueda de directorios.
      2. f: búsqueda de archivos.
    4. -name <"*.txt/xml/py">: se especifica el patron a buscar.
      1. -iname: no distingue entre mayúsculas y minúsculas.
      2. ejemplo: find / -type f -name "*.xml"
      3. ejemplo: find /home -type f -iname user.txt
    5. -user <name user>: enumera todos los archivos de un usuario especifico.
    6. -group <name group user >: enumera todos los archivos de un grupo de usuario especifico.
    7. -size <size>: enumera todos los archivos con un tamaño especifico.
      1. n: coincide con el tamaño exactamente.
      2. +n: coincide con valores mayores al tamaño de n.
      3. -n: coincide con valores menores al tamaño de n.
      4. nc: para bytes.
      5. nk: para KiB.
      6. nM: para MiB.
      7. nG: para GiB.
      8. ejemplo: find /home -type f -size -2k -name "*.txt"
    8. -perm <octal(666) ó simbólica(u=r)>: encuentra todos los archivos con permisos específicos.
      1. -: devolverá archivos con al menos esos permisos configurados.
      2. ejemplo: find /usr/bin -type f -user root -perm -u=s --> encontrar todos los archivos de /usr/bin que son propiedad de root y tienen al menos los el permiso SUID
      3. /: devolvera archivos que los se establescan los permisos.
      4. ejemplo: find /usr/bin -type f -user root -perm /666 --> encontrar todos los archivos de /usr/bin que son propiedad de root y que al menos coincidirá con archivos que sean legibles y escribibles por al menos uno de los grupos (propietario, grupo u otros).
    9. time-related: enumera búsquedas relacionadas con el tiempo.
      1. prefijos para especificar el tiempo.
        1. a: cuando fue el último acceso.
        2. m: cuando fue la última modificación.
        3. c: cuando fue el último cambio.
      2. valor numérico para especificar el tiempo.
        1. +n: especificar que se accedió por última vez hace mas de n.
        2. -n: especificar que se accedió hace menos de n.
      3. -<prefijo>min: enumera archivos que se especifiquen los minutos.
      4. -<prefijo>time: enumera archivos que se especifiquen los dias.
      5. ejemplo: find /usr/bin -type f -mmin -120 --> encontrar todos los archivos de /usr/bin y que se hayan modificado en las ultimas dos horas.
      6. ejemplo: find /usr/bin -type f -atime +10 --> encontrar todos los archivos de /usr/bin a los que no se accedió en las últimos diez días.
      7. 2>/dev/null: suprimir la salida de cualquier error posible para la salida.
      8. ejemplo: find /usr/bin -type f -name "*.txt" 2> /dev/null --> encontrar todos los archivos de /usr/bin con terminación .txt y además que solo aparezca los archivos que se puedan acceder.

  • Asignación de permisos de acceso a directorios o archivos. 
    chmod <permissions> <file>
    1. Se dividen en tres secciones, usuario y grupo y todos los demás. Todo comienza desde el segundo guion, no desde el primero. Tenga en cuenta que todo está en orden secuencial, por lo que los primeros tres caracteres controlan los permisos del usuario, los segundos tres caracteres controlan los permisos del grupo y los tres últimos caracteres controlan los permisos de todos los demás.
    2. -<permissions>: quitar permisos
    3. +<permissions>: agregar permisos
    1. ejemplo: chmod u-x myfile --> quitar el permiso de ejecución al usuario.
    2. ejemplo: chmod ug+w myfile --> agregar permisos de escritura al usuario y al grupo.


    1. ejemplo: Chmod 777 <file> --> dar permiso de ejecución a todos los usuarios.


  • Comando en sistemas derivados de debian para gestionar y administrar los paquetes de distribucion Linux. 
    apt <command>
    1. list : lista los paquetes según los nombres
    2. search: busca en las descripciones de los paquetes
    3. show: muestra detalles del paquete
    4. install: instala paquetes
    5. remove: elimina paquetes
    6. autoremove: Elimina automáticamente todos los paquetes sin utilizar
    7. update: actualiza la lista de paquetes disponibles
    8. upgrade: actualiza el sistema instalando/actualizando paquetes
    9. full-upgrade: actualiza el sistema eliminando/instalando/actualizando paquetes
    10. edit-sources: edita el fichero de información de fuentes
    11. purge <app > : eliminar aplicación
    12. update --fix-missing: error al actualizar los paquetes
